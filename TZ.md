**Мобильное приложение «Безопасный маршрут «Школьная траектория»**

**Основное название:**  
Безопасный маршрут «Школьная траектория».

**Дополнительные названия:**  
Безопасный маршрут «Дом – Школа – Дом».  
Безопасная дорога детства.  
Road Safety.

**Цель проекта:**  
Создание персонального навигатора безопасности для школьников и системы мониторинга для родителей.

**Общая идея приложения:**  
Мобильное приложение с элементами геймификации, направленное на формирование устойчивых навыков безопасного поведения в дорожной среде.  
Приложение анализирует местоположение и поведение пользователя, определяя уровень риска через систему цветовых зон (красная, жёлтая, зелёная), мотивирует его соблюдать правила дорожного движения через систему предупреждения, баллов, бонусов и достижений.

**Цель приложения**

* Повышение осведомлённости о дорожной безопасности.  
* Формирование безопасных поведенческих привычек.  
* Мотивация пользователей через игровые механики.  
* Снижение рискованного поведения в зонах повышенной аварийности.

**Целевая аудитория приложения:**

* Школьники 7-14 лет.  
* Пользователи, участвующие в образовательных программах по БДД.  
* Родители детей.

**Компоненты системы:**

* Мобильное приложение.  
* Аналитический модуль.  
* Система уведомлений.  
* Система бонусов.

**Основные функции приложения:**

* Построение безопасного маршрута.  
* Определение текущего местоположения пользователя.  
* Анализ контекста дорожной обстановки.  
* Отображение зоны риска: красная, жёлтая, зелёная.  
* Контроль отклонений.  
* Push-уведомления о рисках.  
* Критическая кнопка (SOS): мгновенная передача координат и аудио- видеозаписи окружения родителям при отклонении от маршрута или активации ребенком.  
* Начисление/списание баллов.  
* Сбор статистики.  
* История активности.  
* Экран достижений: баллы, уровень, достижения, бонусы, статистика.  
* Профиль пользователя.  
* Админ панель.

**Цветовые зоны риска:**

* Красная — опасное поведение, нарушение правил, высокий уровень риска. **Действие**: предупреждение пользователя и родителей; бонусные баллы не начисляются, бонусные баллы списываются, снижение рейтинга, регресс в достижениях.  
* Жёлтая — потенциальный риск, подозрение на небезопасное поведение, вход в зону повышенной аварийности, зафиксированы поведенческие факторы риска. **Действие**: предупреждение пользователя, бонусные баллы не начисляются.  
* Зелёная — безопасное поведение, пользователь соблюдает правила, нет опасных манёвров, пользователь находится в разрешённых/безопасных местах, отсутствуют признаки риска. **Действие**: пользователь получает бонусные баллы, накопление очков, рост рейтинга, прогресс в достижениях.

**Система мотивации:**  
**Баллы:**

* Начисляются за безопасное поведение.  
* Снижаются при нарушениях.  
* Используются для формирования рейтинга.

**Уровни:**

* Новичок  
* Ответственный участник.  
* Эксперт безопасности.  
* Амбассадор безопасности.

**Достижения:**

* «7 дней без нарушений».  
* «1000 безопасных метров».  
* «Безопасный маршрут недели».

**Бонусы:**

* Виртуальные награды.  
* Цифровые сертификаты.  
* Потенциальная интеграция с партнёрами (скидки, призы).

**Игровые элементы:**

* Аватар (маскот) «Северный олень» или «Белый мишка», меняющий состояние в зависимости от безопасности.

**Пользовательский сценарий:**

* Пользователь устанавливает приложение.  
* Предоставляет разрешение на геолокацию.  
* Родители пользователя (админ) устанавливают админ пароль на вход в админ панель приложения ребёнка.  
* Родители пользователя (админ) в админ панели приложения ребёнка или в своём приложении указывают из записной книжки кому направлять уведомления. Данные двух приложений синхронизируются.  
* Родители пользователя (админ) в админ панели приложения ребёнка или в своём приложении указывают на карте безопасные маршруты и зоны риска. Данные двух приложений синхронизируются.  
* Пользователь видит текущую зону (цветовой индикатор).  
* Родители пользователя (админ) в своём приложении видят местоположение пользователя на карте.  
* Пользователь и родители пользователя (админ) получают уведомления при изменении уровня риска.  
* Пользователь набирает баллы, рейтинги, достижения, бонусы за безопасное поведение, теряет их за не безопасное поведение.

**Архитектурные требования:**

* Android (Kotlin). При возможности и на iOS.  
* Поддержка актуальных версий Android.  
* Работа с разрешениями на геолокацию.  
* Использование архитектуры MVVM.  
* Поддержка масштабирования функционала.  
* Возможность подключения аналитики.

**Перспективы развития:**

* Интеграция с государственными системами «Умный город». Службам города: отслеживание детей на карте и сопоставление с картинкой на камерах. Детям: предупреждения о дорожных условиях. Родителям: возможность посмотреть в реальном времени передвижение ребёнка на ближайшей общедоступной камере.  
* Подключение школ.  
* Рейтинги между пользователями.  
* Командные челленджи.  
* Расширенная аналитика.  
* Интеграцией с «Морошкой»: система API для передачи накопленных за безопасную ходьбу баллов в региональную экосистему бонусов. В последствии и с другими платформами (магазины брендированной одежды Ямал, «Сбер Спасибо», сети ресторанов быстрого питания, и т.п.).  
* Интеграция элементов игры-симулятора «Дорогами Ямала». Возможна сетевая многопользовательская игра (предположение: в виде web странички внутри приложения).

**Технические требования:**

* GPS в реальном времени. Можно в заданный родителями интервал времени (подстройка под часы передвижения в школу и обратно) делать точный режим, в остальное время – менее точный режим – для оптимизации энергопотребления.  
* Учёт погрешностей координат, в том числе в условиях низких температур, снегопадов и ветров.  
* Офлайн-карты: Базовая навигация должна работать при нестабильном интернете (актуально для отдаленных районов)  
* Оптимизация энергопотребления.  
* Уведомления Critical Alerts (пробивающие режим «Не беспокоить») при приближении к желтым (пешеходные переходы) и красным зонам (дороги) при включенном экране телефона.  
* Звуковое уведомление (прерывание системных звуков, включение уведомления о приближении к опасным зонам).

**Ожидаемый результат:**

* Мобильное приложение.  
* Система анализа маршрутов.  
* Отчёты для родителей.  
* Отчёты для школ (обезличенная информация для контроля или для корректировки информации по маршрутам).  
* Геймификация.  
* Отчёты (логи) для внешних систем учёта бонусов.
